2 上传文件要求

.docx 首选

包含 Word 内置目录，章节层级识别最准确。标题使用「标题 1/2/3/4」样式或标准编号前缀（一、1. (1) ①）效果最佳。

.pdf 可检索 PDF

需可用鼠标选中文字（非扫描件）。标题字号需明显大于正文，或带标准数字编号前缀。

.doc 旧格式

需服务器安装 LibreOffice 自动转换，建议提前另存为 .docx 格式后上传。

💡

生成质量的关键在于原文内容充实度——章节原文越详细（含技术指标、功能描述、约束条件），AI 扩写效果越好。仅有标题、无正文的章节将被自动跳过。

5 常见问题

章节识别数量为 0 或很少？ ▸
DOCX：检查标题是否使用「标题 1/2/3/4」样式，或包含标准编号前缀（一、1. (1) 等）。
PDF：确认 PDF 可检索（鼠标可选中文字），且标题字号明显大于正文。
很多章节显示为「已跳过」？ ▸
说明该章节在规范书中只有标题、无正文内容。系统不会无中生有，Word 文档中会保留标题，结构与规范书一致。
生成中途断开，已完成内容会丢失吗？ ▸
不会。已完成章节保存在服务器内存中，可直接点击「下载 Word」导出已完成部分。
豆包 API 提示模型不存在？ ▸
豆包（火山引擎）需在控制台创建推理接入点，模型 ID 格式为 ep-xxxxxxxx-xxxxx。在配置面板「模型」字段填写接入点 ID。
